## 2. 핵심 기능 상세 분석

### 2.1 프로젝트 관리
- **프로젝트 생성**: 이름, 설명, 평가방법 선택
- **프로젝트 상태 4단계**:
  1. **생성중**: 모델 구성요소(기준, 대안) 및 평가자 구성 중
  2. **대기중**: 모델 완료, 평가자 배정 가능, 평가 시작 전
  3. **평가중**: 평가 진행 중 (모델 수정 시 데이터 삭제)
  4. **평가종료**: 평가 종료, 결과 확인만 가능
- **프로젝트 기능**: 관리, 수정, 삭제, 중단/완료, 워크숍 진행, 복사

### 2.2 평가방법 (3가지)
| 평가방법 | 코드값 | 설명 | 용도 |
|----------|--------|------|------|
| 쌍대비교-실용 | 12 | 최소한의 쌍대비교 (기본값) | 실무 활용 |
| 직접입력 | 20 | 데이터 직접 입력으로 중요도 도출 | 데이터 기반 |
| 쌍대비교-이론 | 10 | 모든 쌍대비교 실시 | 논문 작성 |

### 2.3 모델 구축
- **기준(Criteria) 관리**:
  - 기준 추가/수정/삭제
  - 하위기준 추가 (계층구조)
  - 기준별 평가방법 개별 설정 가능
- **대안(Alternative) 관리**:
  - 대안 추가/수정/삭제
  - 하위대안 추가 가능
  - 기준만의 priority 도출도 가능 (대안 생략)
- **평가자(Participant) 관리**:
  - 평가자 추가/수정/삭제
  - 이메일, 이름, 전화번호 입력
  - 평가자 배정 (체크박스 선택 후 배정)
  - 배정된 평가자는 파란색 표시

### 2.4 쌍대비교 (Pairwise Comparison)
- **9점 척도**: 동등(1) ~ 극히(9) 중요
- **비교 방식**: 두 항목 중 어느 쪽이 얼마나 중요한지 선택
- **비일관성비율(CR)**: 자동 계산, 0.1 초과 시 빨간색 경고
- **판단 도우미**: 비일관성 수정을 위한 제안 (1, 2, 3순위)
- **재평가**: 빨간색 비일관성비율 클릭으로 해당 매트릭스 이동

### 2.5 직접입력
- 데이터 값을 직접 입력하여 중요도 도출
- **역수 변환**: 값이 낮을수록 좋은 경우 (가격, 비용 등) 역수 적용

### 2.6 결과 확인
- **차트 + 표** 형태로 결과 표시
- **종합 중요도**: 대안/기준의 통합 중요도
- **세부 중요도**: 기준별, 대안별, 평가자별 평가값
- **비일관성비율**: 각 매트릭스별 CR 표시
- **Excel 저장**: 결과 다운로드 기능
- **결과 접기/펼치기**: 붉은색 네모칸 클릭

### 2.7 부록 기능
#### 브레인스토밍
- 프로젝트 생성 직후 사용 (기준 추가 전)
- 대안 → 장단점 → 판단기준 순으로 입력
- Drag&Drop으로 계층구조 구성
- 모델 보기 → 모델 확정

#### 그룹별 가중치 산출
- 평가자별 가중치 조정
- 일부 평가자만 통합 결과 산출
- 결과는 DB 미저장 (Excel 별도 저장 필요)

#### 민감도 분석
- 기준의 중요도 변경 시 대안 결과 변화 확인
- 상위기준 선택 → 변경값 입력 → 분석 시작
- 결과 서버 미저장

#### 워크숍
- 모든 평가자가 한자리에서 동시 평가
- 관리자 화면: 실시간 동기화(on/off), 이전/다음 제어
- 평가자 화면: 관리자 제어에 따라 자동 변경
- 직접입력 워크숍도 지원

#### 자원배분 (추가기능)
- 한정된 예산을 각 대안에 최적 배분
- 비용 입력 → 예산총액 입력 → 계산
- 일정 비율 증감 시뮬레이션

## 4. 구현 시 참고 기술 사항

### 4.1 원본 기술 스택 (I Make It)
- **서버**: ASP.NET (*.aspx, *.asmx)
- **클라이언트**: jQuery 1.11.0
- **API**: SOAP Web Service (makeit.asmx)
- **세션 관리**: sessionStorage
- **접속키 방식**: URL 파라미터 (makeit=TOKEN_USERNAME)

### 4.2 데이터 모델 (추정)
```
Project (프로젝트)
├── id, name, description, type (평가방법), state (상태)
├── Criteria (기준) - 트리 구조
│   ├── id, name, description, eval_method
│   └── Sub-Criteria (하위기준)
├── Alternative (대안)
│   ├── id, name
│   └── Sub-Alternative (하위대안)
├── Participant (평가자)
│   ├── id, email, name, tel
│   └── assigned (배정여부)
└── Evaluation (평가 데이터)
    ├── pairwise_comparison (쌍대비교 값)
    ├── direct_input (직접입력 값)
    └── result (계산 결과: priority, CR 등)
```

### 4.3 API 엔드포인트 (추정)
- `makeit.asmx/check_manager` - 관리자 세션 체크 (5분 간격)
- 프로젝트 CRUD
- 기준/대안 CRUD
- 평가자 관리
- 평가 데이터 저장/조회
- 결과 계산 (AHP 알고리즘)
- Excel 다운로드
