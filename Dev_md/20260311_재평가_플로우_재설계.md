# 재평가 플로우 재설계 (평가자 관점)

**작업일**: 2026-03-11

---

## 변경 전 문제

1. "재평가하기" 클릭 후 결과 페이지에 머물면서 상태 꼬임
2. SignaturePanel이 "평가가 완료되었습니다" 유지 → "평가 완료" 버튼 안 보임
3. 설문 수정 → 평가 수정 → 재완료 의 자연스러운 흐름 없음
4. 안내 패널만 표시되고 실제 네비게이션이 즉시 일어나지 않음

---

## 새 플로우 (평가자 관점)

```
결과 페이지 (완료 상태)
  │
  ▼ "재평가하기" 클릭 → 확인 다이얼로그
  │
  ├─ 설문이 있는 경우 ─────────────────────┐
  │  설문 수정 페이지 (기존 답변 로드)      │
  │  [설문 수정 건너뛰기] [설문 수정 후 평가하기]
  │        │                    │
  │        └────────┬───────────┘
  │                 ▼
  ├─ 설문이 없는 경우 ─┐
  │                    ▼
  │  쌍대비교 평가 페이지 (기존 비교값 유지)
  │  비교값 수정 → "결과 보기"
  │                 │
  │                 ▼
  │  결과 페이지 (미완료 상태)
  │  - "평가 완료" 버튼 활성화
  │  - 비일관성비율 탭 클릭 이동 가능
  │  - 재점검 필요 항목 섹션 표시
  │                 │
  │                 ▼
  └─ "평가 완료" 클릭 → 재잠금
```

---

## 변경 파일

### 1. `src/pages/EvalResultPage.jsx`

- `handleUnlock`: 확인 후 **즉시 페이지 이동** (결과 페이지에 머물지 않음)
  - 설문 있음 → `/eval/project/:id/pre-survey?edit=1&next=eval`
  - 설문 없음 → `/eval/project/:id` (쌍대비교 평가)
- 확인 다이얼로그 메시지: 설문 유무에 따라 분기
  - "설문 응답부터 다시 수정합니다" / "쌍대비교 평가를 수정합니다"
- `unlocked` 상태 및 안내 패널 제거 (페이지를 떠나므로 불필요)
- SignaturePanel `key={hasSigned ? 'signed' : 'unsigned'}` 유지
  → 결과 페이지 재진입 시 리마운트되어 "평가 완료" 버튼 표시

### 2. `src/pages/EvalPreSurveyPage.jsx`

- `next` 쿼리 파라미터 처리 (`searchParams.get('next')`)
- 설문 제출 후 이동 분기:
  - `?edit=1` (결과에서 설문 수정) → 결과 페이지로 복귀
  - `?edit=1&next=eval` (재평가 플로우) → 평가 페이지로 이동
- 버튼 텍스트:
  - 재평가: "설문 수정 후 평가하기"
  - 일반 수정: "설문 수정 완료"
- 취소 버튼:
  - 재평가: "설문 수정 건너뛰기" → 평가 페이지로 바로 이동
  - 일반 수정: "취소" → 결과 페이지로 복귀

### 3. `src/pages/EvalResultPage.module.css`

- unlockGuide 관련 스타일 전부 삭제 (미사용)

## 핵심 포인트

- **페이지 이동 기반**: 결과 페이지에서 상태만 바꾸지 않고, 실제 페이지를 이동
  → 상태 꼬임 문제 원천 해소
- **`replace: true`**: 히스토리 스택에서 결과 페이지를 교체
  → 뒤로가기 시 완료 상태의 결과 페이지로 돌아가지 않음
- **SignaturePanel 리마운트**: 결과 페이지 재진입 시 DB 재조회
  → 서명 없음 확인 → "평가 완료" 버튼 정상 표시
