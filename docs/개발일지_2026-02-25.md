### 2. [P0] ToastContext useCallback → useMemo 버그 수정

**문제**: `useCallback`은 함수 전용 메모이제이션 훅인데, 객체에 적용하고 있었음. React API 오용.

```javascript
// Before (잘못된 사용)
const toast = useCallback({
  success: (msg) => addToast(msg, 'success'),
  error: (msg) => addToast(msg, 'error'),
}, [addToast]);

// After (올바른 사용)
const toast = useMemo(() => ({
  success: (msg) => addToast(msg, 'success'),
  error: (msg) => addToast(msg, 'error'),
}), [addToast]);
```

**수정 파일**: `src/contexts/ToastContext.jsx`

### 3. [P0] 관리자 권한 DB 기반 전환

**문제**: 관리자 판별이 소스 코드에 하드코딩된 이메일 배열(`ADMIN_EMAILS`)에 의존.
번들에 포함되어 전세계에 공개되며, 관리자 추가 시 코드 수정+재배포 필요.

**수정 내용**:
- `ADMIN_EMAILS` 배열 완전 제거
- `user_profiles.role === 'admin'` DB 컬럼 기반으로 전환
- `profileLoading` 상태 추가 — 프로필 로드 완료 전 잘못된 리디렉션 방지

```javascript
// Before
const ADMIN_EMAILS = ['aebon@kakao.com', ...];
const isAdmin = isLoggedIn && ADMIN_EMAILS.includes(state.user?.email);

// After
const isAdmin = isLoggedIn && state.profile?.role === 'admin';
```

**수정 파일**: `src/contexts/AuthContext.jsx`

### 4. [P1] AdminGuard 관리자 라우트 보호 적용

**문제**: `AdminGuard` 컴포넌트가 정의만 되어 있고 어떤 라우트에서도 사용되지 않음.
모든 `/admin/*` 경로가 `ProtectedRoute`(로그인만 확인)로만 보호되어, 일반 사용자도 URL 직접 입력으로 관리자 페이지 접근 가능.

**수정 내용**:
- `AdminGuard`에 `profileLoading` 체크 추가 (비동기 프로필 로드 대기)
- `/admin/*` 모든 라우트(11개)를 `ProtectedRoute` → `AdminGuard`로 교체
- 비관리자 접근 시 홈(`/`)으로 리디렉션

**수정 파일**: `src/App.jsx`, `src/components/common/AdminGuard.jsx`
