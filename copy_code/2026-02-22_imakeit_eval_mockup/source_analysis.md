# 평가 UI 목업(Eval Mockup) 페이지 소스 분석

## 페이지 정보
- **타입**: 사용자 제작 ASP.NET 목업 (CodeBehind: default.aspx.cs)
- **역할**: 평가 진행 화면의 UI 디자인 참고용 목업

## UI 구성 요소

### 1. 모드 전환
- "현재 : 평가자 MODE" / "관리자 MODE로 이동" 버튼
- switchMode() JS로 텍스트 토글

### 2. 평가 정보 헤더
- 제목: "평가진행 시작하기 ( 평가중 )"
- 프로젝트: "대학원생의 연구역량"
- 안내: 1차 평가기준 간 상대적 중요도 판단

### 3. 기준별 중요도 바 차트
| 기준 | 색상 | CSS 클래스 | 중요도 |
|------|------|-----------|--------|
| 창의력 | #2980b9 (파랑) | bar-c | 15.308% |
| 분석능력 | #2ecc71 (초록) | bar-a | 32.075% |
| 기술능력 | #f39c12 (주황) | bar-t | 20.628% |
| 문제 해결능력 | #c0392b (빨강) | bar-p | 31.988% |

### 4. 진행 상태
- 문항: 1/6
- 완료: 5/6 문항
- 비일관성비율: 1.35222
- "타당성 검증 기준을 통과하였습니다!!"
- 경고: "평가를 완료하지 못했습니다. (5/6)"

### 5. 판단 도우미 영역
- 5개 추천 버튼 (1~5)
- highlightHelperStep(n) → alert로 추천 셀 안내
- 안내 사항: 추천 셀 참고, 기존/추천 셀 선택, 세밀 조정

### 6. 평가 방법 안내
- 상대적 중요도 선택 방법
- 재클릭 선택 취소
- 실시간 저장

### 7. 하단 버튼
- "모델보기" / "프로젝트 전체 모델보기"

## 디자인 특징 (React 참고)
- 모던 카드 레이아웃 (max-width: 900px, border-radius: 10px)
- 플렉스 기반 바 차트 (criteria-row → flex)
- 색상 팔레트: flat design 계열 (#2980b9, #2ecc71, #f39c12, #c0392b)
- 부드러운 그림자: box-shadow: 0 2px 10px #ddd
- 도우미 영역: 구분된 배경 (#f4fafc)
